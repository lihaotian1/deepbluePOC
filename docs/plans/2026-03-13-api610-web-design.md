# API610询价文件智能比对 Web 设计文档

## 1. 目标

构建一个前后端分离系统，支持上传采购商询价 PDF，按章节切分并可编辑内容，再逐段对 `data/知识库/标准化配套知识库.json` 做流式比对，输出匹配条目与类型（P/A/B/C），并支持 Excel 导出。

## 2. 核心流程

1. 用户上传 PDF。
2. 后端调用现有 `chapter_splitter.py` 的工程化切分方法生成有序章节块。
3. 前端展示章节块（Markdown 预览 + 可编辑）。
4. 用户点击“比对知识库”后，后端按章节流式处理：
   - 先判断章节属于哪些知识库分类 key。
   - 若无分类命中，输出“其他”。
   - 若有分类命中，再判断该分类内哪些条目与章节语义一致。
   - 若无条目命中，输出“其他”；有命中则输出条目文本和类型（P/A/B/C）。
5. 前端实时显示处理进度和结果。
6. 比对完成后导出 Excel。

## 3. 关键设计决策

- LLM 接口：采用 OpenAI 兼容模式。
- 匹配口径：采用“语义一致”。
- 实时交互：后端 SSE 推送逐段比对结果。
- 会话管理：后端内存会话存储（以 `doc_id` 关联切分内容、编辑内容、比对结果）。

## 4. 模块划分

- 后端（FastAPI）
  - 文档上传与切分 API
  - 章节编辑回写 API
  - SSE 流式比对 API
  - Excel 导出 API
  - 知识库加载与标准化模块
  - OpenAI 兼容 LLM 调用模块

- 前端（React）
  - 上传组件
  - 章节卡片（编辑/预览）
  - 比对控制与流式状态展示
  - 结果标签展示（分类、P/A/B/C、其他）
  - 导出按钮

## 5. 非功能要求

- 页面风格“美观大气”，同时兼顾桌面端与移动端。
- 项目可通过 Docker Compose 一键部署。
- 关键能力提供基础测试覆盖（知识库解析、匹配流程、导出服务）。
