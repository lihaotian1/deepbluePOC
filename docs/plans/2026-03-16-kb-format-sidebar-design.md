# 双格式知识库与主页/侧边栏优化设计文档

## 1. 目标

在现有系统上继续增强三部分能力：

1. 兼容两种知识库源格式：
   - 结构化格式 `{分类: [{条目: 值}]}`
   - 扁平 key-value 格式 `{key: value}`
2. 支持知识库文件级新建/删除、搜索、分页。
3. 优化主页文案、侧边栏 logo 排版，以及章节卡片左右内容区行为。

## 2. 知识库格式兼容

- 后端在读取 JSON 文件时自动识别格式：
  - 若 value 为列表，则识别为 `grouped`
  - 若 value 为字符串，则识别为 `flat_key_value`
- 对前端统一返回“分类树”模型：
  - `grouped` 维持原样
  - `flat_key_value` 转换为：每个 key 作为一个分类，每个分类下仅一个条目，条目文本为原 value
- 保存时根据 `format` 字段还原原始格式，保证 `API 610通用标准.json` 不影响另外两个知识库。

## 3. 知识库管理页

- 顶部工具区新增：
  - 新建知识库文件
  - 删除当前知识库文件
  - 搜索框
- 搜索针对当前已打开知识库执行：
  - `grouped`：分类名、条目文本、值
  - `flat_key_value`：key（分类名）和 value（唯一条目文本）
- 分页统一为每页 10 个分类卡片。
- `flat_key_value` 文件在页面上仍按分类树显示，但其条目值输入区隐藏，仅保留文本编辑区。

## 4. 侧边栏与 Logo

- Logo 置顶、放大、居中展示。
- `智能比对` 文字置于 logo 下方。
- 收起/展开按钮放在 `智能比对` 文字右侧。
- 收起后保留居中 logo 与一级菜单图标。

## 5. 主页文案与交互调整

- 顶部只保留 `AI Assistant` 标识，删除主标题和描述文案。
- 上传区文案调整：
  - `上传询价文件` → `上传文件`
  - `选择询价文件` → `选择文件`
- 删除上传区、比对区、筛选区的说明性描述文案。

## 6. 章节卡片左侧原文区

- 左侧原文区取消滚动与拖拽放大。
- 编辑态使用自动增高 textarea，高度随内容变化，最小高度为一行。
- 预览态默认完整展示所有内容，不截断、不滚动。
- 卡片左右两栏保持等高，由外层网格拉伸实现。
