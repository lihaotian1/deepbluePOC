# 侧边栏与知识库管理设计文档

## 1. 目标

在现有询价比对系统中增加统一左侧导航，并新增“知识库”管理页面，支持从 `data/知识库/*.json` 读取知识库文件，在前端做结构化增删改查，保存后同步回写 JSON 文件。

## 2. 布局设计

- 页面采用 `左侧导航 + 右侧主内容` 布局。
- 左侧导航展开时约占页面宽度 15%，显示 logo、菜单图标与文字。
- 左侧导航收起后仅显示 logo 缩略与菜单图标。
- 菜单包含：`主页`、`知识库`。
- `知识库` 菜单下有二级菜单，列出 `data/知识库` 下所有 JSON 文件。

## 3. 主页改造

- 主页功能保持不变，仍包含上传、切分、流式比对、类型筛选、Excel 导出。
- 原有页面内容迁移到统一布局的主内容区中。

## 4. 知识库页面设计

- 使用结构化编辑，不直接暴露原始 JSON 文本。
- 每个 JSON 文件对应一个知识库。
- 页面按 `分类 -> 条目` 结构展示。
- 支持以下操作：
  - 新增分类
  - 分类重命名
  - 删除分类
  - 新增条目
  - 编辑条目文本与值
  - 删除条目
- 保存时保持 JSON 结构为 `{分类: [{条目: 类型}]}`。

## 5. 后端设计

- 新增知识库管理服务，负责：
  - 列出知识库文件
  - 读取指定 JSON 文件并转成结构化响应
  - 保存结构化数据并回写 JSON
- 新增 API：
  - `GET /api/v1/knowledge-bases`
  - `GET /api/v1/knowledge-bases/{file_name}`
  - `PUT /api/v1/knowledge-bases/{file_name}`
- 保存默认比对知识库文件后，自动刷新内存中的 matcher 知识库缓存。
- 暴露 logo 静态资源，供前端侧边栏显示。

## 6. 前端设计

- 新增 `AppShell` 容器统一管理：当前页面、侧边栏折叠状态、知识库文件列表。
- 新增 `Sidebar` 组件显示 logo、一级菜单、知识库二级菜单。
- 新增 `KnowledgeBasePage` 页面实现结构化知识库编辑。
- 前端知识库编辑先操作本地状态，再统一保存到后端。

## 7. 风险与约束

- JSON 文件名可能包含中文，接口需使用 URL 编码。
- 保存时需限制访问范围，仅允许操作 `data/知识库` 目录内的 JSON 文件。
- 当前比对逻辑默认仍使用配置中的主知识库文件，不新增“比对时切换知识库”的能力。
